# Client User Manual
This manual discuss how a client may interact with the webpage structure of this project. A client for the purposes of this manual is any person who is interested in submitting any form to Danville Planning and Zoning. 

A client must make an account to be able to intearct with the system. All interactions start the login page. From here a new client may choose to make a new account by the "Create Account" Button, which asks for a new username and password.

From the login page the user will be directed to the client dashboard page. There are four notable sections: the top navbar, the Quick Actions section, the Submitted Forms tab, and saved drafts. From this page a client may decide what actions they want to do. Details for each of these sections are given below.


## Navbar
The navbar contains two buttons: "Manage Account" and "Logout". The logout button simply removes current session information and redirects back to the login page. 


### Manage Account Page
Manage Account directs the user to a page titled "Manage Your Account". On this page the user may change their saved email or their password. Both options require retyping for confirmation before changing the attribute.

Finally, there is an option to delete the account. This also requires confirmation before deletion. 

## Quick Actions and Submit a New Form
Quick actions currently only contains the option Submit New Form. 

This button navigates to another page that allows the user to pick the formt they wish to edit. There is a drop down menu that displays all form options including Administrative Appeal Requests, Adjacent Property Owner Forms, and various applications. The user ought to scroll to the appropriate form and click the Open Form button.

### Form editing
Each form contains fields that must be filled out and some optional forms. Each form has individual charactersitics that ought to be considered when filling out the form. In particular, the user ought to be aware that many forms have buttons to add additional applicants, appellants, or property owners. This allows the user to add all relevant parties to the form. 

Once the form is filled out, there is a button at the bottom of the page displayed as "Submit Application", which allows the user to submit the form to be saved in the database. In the current version, some forms may contain an additional button to save the form. This saves all currently written information to the database, but does not display it to the government workers or departments.

## Submitted Forms
This section displays all forms in a table that the client has submitted and a summary of their status. The form details can viewed by clicking the "View Details" button on the right side of the table. This button displays the form status as well as all information that was submitted in the form. 

The page to view forms also has a button on its navbar to print the page as a pdf as necessary, formatted like the page.

## Saved Drafts
The final section is the saved drafts, which allows the user to click edit draft and edit previously saved information. This opens a page with prefilled in information that was saved and allows any desired changes to be made. The page that is displayed is other aspects identical to the page to submit a new form and once edited, the form may be saved or finally submitted.
